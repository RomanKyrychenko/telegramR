# SetClientDHParamsRequest

Telegram API type SetClientDHParamsRequest

Methods: - new(nonce, server_nonce, encrypted_data): create new
instance. - to_list(): return an R list representation. - to_raw():
serialize to raw vector (bytes) in little endian as used in TL. -
from_reader(reader): class method, construct instance from a \`reader\`
object that exposes \`read_large_int(bits = 128)\` and
\`tgread_bytes()\`.

Note: 128-bit integer handling is represented here as numeric/double
placeholders. For precise 128-bit two's-complement behavior, use a
big-integer library and implement a precise serializer.

Represents the client's response during the Diffie-Hellman key exchange.
Contains the client's `nonce`, the server's `server_nonce`, and the
`encrypted_data` payload sent to the server for verification.
